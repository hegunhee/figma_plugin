MCP Bridge 사용 가이드

MCP Bridge란?

Claude Code에서 Figma 디자인 데이터를 직접 조회할 수 있는 브릿지입니다. 이 플러그인이 실행 중이면 Claude Code가 아래 도구들을 사용할 수 있습니다.

사용 가능한 도구

figma_get_node

특정 노드 ID로 디자인 스펙 조회. URL의 node-id 값 사용.

figma_get_selection

현재 선택된 노드들의 디자인 스펙 조회.

figma_get_page_nodes

현재 페이지의 최상위 노드 목록 조회.

figma_get_file_info

파일명, 페이지 목록, 현재 페이지 정보.

figma_get_comments

파일의 댓글 조회. 미해결 댓글만 필터 가능.

figma_analyze_comments

미해결 댓글 + 각 위치 노드 스펙을 한번에 조회.

figma_reply_comment

댓글에 답글 달기. 수정 완료 내역 등을 남길 때 사용.

figma_export_node

노드를 PNG/SVG/JPG 이미지로 내보내기 (base64).

figma_get_styles

로컬 디자인 시스템 스타일 (색상, 텍스트, 이펙트) 조회.

Detail Level

detail 파라미터로 응답 상세도를 조절합니다.

  • minimal id, name, type, size만
  • standard 레이아웃, 색상, 타이포 포함 (기본값)
  • full 디자인 토큰, 컴포넌트 속성, 오버라이드 전체

활용 워크플로우

  • 디자인 → 코드 구현: get_node → 스펙 추출 → 코드 생성
  • 디자인 QA: get_selection → 코드 값과 1:1 비교
  • 댓글 기반 수정: analyze_comments → 문제 파악 → 수정 → reply_comment
  • 디자인 토큰 추출: get_node(detail:"full") → boundVariables 확인

테스트 버튼

플러그인 하단의 버튼으로 각 도구를 직접 테스트할 수 있습니다. 결과는 Activity Log에 표시됩니다.

문제 해결

  • 빨간 점: Claude Code가 실행 중인지 확인 (MCP 서버 자동 시작)
  • 노란 점: 재연결 시도 중. 잠시 대기하거나 "재연결" 버튼 클릭
  • 도구 파라미터 안 보임: cd mcp-server && npm run build 후 Claude Code에서 /mcp 재접속
